## TL;DR
The DO-IT relay baton fails silently from three small, real writer/reader defects: rev's
skill has no baton field template (so it emits `updated_at:` not `handed_off_at:` → F11),
orc/rev arming omits `CWD=`, and the reader doesn't unquote `baton_pane`. The proportionate
fix is to give rev a literal template (like orc already has), emit `CWD=` in arming, unquote
in the reader, lightly harden the reader's already-present protections, and convert the
silent log-skip into a loud-but-rate-limited alert plus a dark-role stall guard. No new CLI,
no state machine, no `/goal` — those were over-built for a one-box, one-operator setup that
the operator actively watches. Then bump the public release to 3.8.0 and port to AS.

## Why this exists (the three stable defects + silence)
- **Incident 1 — field-name (F11).** rev's skill (`skills/rev/SKILL.md:~63`) gives only a
  prose handoff instruction, no field template, so rev emits `updated_at:`; the reader
  requires `^handed_off_at:` and silently skips. Historical: ~42h / ~1,426 skip lines. orc
  has a template and never hit this — the fix is presence of a template.
- **Incident 2 — CWD omission.** `skills/{orc,rev}/SKILL.md:55/34` arm with `PANE` only;
  the reader's git-root fallback silently exits off-tree. (orc's running `/tmp/orc-active`
  was hand-patched; the skill source still regresses on next boot.)
- **Incident 3 — pane-quoting.** orc writes `baton_pane: "%0"`; reader `awk '{print $2}'`
  (L93) yields `"%0"`, compared `"%0" != %0` (L94) → mismatch skip.
- **The common harm is silence** (a log line nobody watches) and **no stall signal** when a
  role's relay stays dark — that's how a defect ran ~42h unnoticed.

## Reader baseline (already in production — keep, don't rebuild)
`orc-relay-watch.sh` already has: `flock` single-consumer (L36-37), freshness gate
`FRESH_SECS=5400` (L77-81), mtime consume-once (L84-89/142), dead-pane check (L99-103),
status head-scan (L68). This release hardens two thin gaps and adds loudness.

## Requirements

**R1 — Give every role a literal baton template; emit CWD in arming.**
- Required behaviour: rev (and watcher, if lacking) gets a fenced baton template block with
  the exact required fields including `handed_off_at:`, matching orc's. All three arming
  lines emit `printf "PANE=%s\nCWD=%s\n" "$TMUX_PANE" "$(pwd)"`.
- Acceptance:
  - [ ] `grep handed_off_at ~/do-it/skills/rev/SKILL.md` matches inside a fenced template.
  - [ ] All three skills' arming lines include `CWD=`.
  - [ ] A fresh rev session, following its skill, writes a baton the reader accepts (no
        `incomplete (no handed_off_at)` skip).
- Severity: P0 (F11 root).

**R2 — Reader: unquote, atomic marker, collision-proof consume key.**
- Required behaviour: (a) unquote `baton_pane`/`PANE` in the reader (`tr -d '"'` or
  equivalent) so quoted and unquoted compare equal; (b) write the consume marker
  atomically (tmp-then-rename, not the direct write at L142); (c) make consume-once
  collision-proof — add a short `baton_id` (one `uuidgen` in the template) to the baton
  header and key consume-once on it instead of bare mtime (mtime can collide within a
  second). Keep flock, freshness, dead-pane as-is.
- Acceptance:
  - [ ] `baton_pane: "%0"` and `baton_pane: %0` both relay correctly to pane `%0`.
  - [ ] Two batons touched in the same second are not conflated (distinct `baton_id`).
  - [ ] A partially-written marker is never read as consumed (atomic rename proven).
- Severity: P0/P1.

**R3 — Loud-but-rate-limited failure through a NAMED channel.**
- Required behaviour: name the real channel (the tmux/iTerm bell the operator already gets,
  plus a watched `/tmp/{role}-relay-error` sentinel). On a HANDED-OFF-but-invalid baton OR a
  resolution failure (no CWD, dead pane), write the sentinel and ring the bell ONCE per
  error fingerprint; suppress repeats while the sentinel exists. Recovery is `rm` of the
  sentinel (document it; no CLI).
- Acceptance:
  - [ ] An invalid HANDED-OFF baton rings the bell once and is not re-alarmed every minute
        (verified by a second tick producing no new notification).
  - [ ] A well-formed baton relays with no alarm.
- Severity: P1.

**R4 — Dark-role stall alert.**
- Required behaviour: if a baton is HANDED-OFF and older than `2 × FRESH_SECS` (≈3h) and
  still unconsumed, write `/tmp/{role}-relay-stall`, ring the bell once, and stop — so a
  dark relay surfaces in hours, not after 42h. (This is the proportionate replacement for
  rev2's durable attempt-counter: a stall signal, not a state machine.)
- Acceptance:
  - [ ] A synthetic 3h-old HANDED-OFF baton produces a stall sentinel + one bell; a fresh
        one does not.
- Severity: P1.

**R5 — Release hygiene (§7) + distribution.**
- Acceptance:
  - [ ] `~/do-it/DO-IT.md:3` → 3.8.0; `docs/DESIGN.md` dated decision (right-sized relay
        hardening; CLI + `/goal` considered and rejected for scale) AND the stale
        `## Roles — 3 (was 5)` section updated to include `rev` + `watcher`.
  - [ ] `CHANGELOG.md` `[3.8.0]` links this doc; README relay/role-map touched; tag `v3.8.0`.

**R6 — Port into the AS instance.**
- Required behaviour: port R1–R4 to `/opt/albert-scott` (`~/.claude/skills/{orc,rev,watcher}`,
  `scripts/orc-relay-watch.sh`), keep the relay hook in `.claude/settings.local.json`,
  reconcile AS `DO-IT.md` → 3.8.0. Preconditions: verify no orc owns the tree before
  touching it; port the reader + skills, settings.local.json last; verify against the REAL
  live `docs/sessions/{orc,rev}-relay.md`. Never run `setup.sh` here. (The right-sized fix
  makes this a small, safe port — the live orc-relay deadlock is relieved here.)
- Acceptance:
  - [ ] AS carries R1–R4; orc + rev relays both fire on real live batons; AS `DO-IT.md`
        reads 3.8.0; the port did not collide with a live orc.
- Severity: P0 for live AS relief.

## Considered & rejected (with the round-2 evidence, so this isn't reopened)
- **Writer CLI / typed state machine (rev2):** rejected — single-CLI SPOF across all roles,
  bootstrap chicken-egg on the port, RELAYED-before-send race, attempt-counter void on
  crashed boot; and it can't enforce LLM use of the CLI without yet more machinery. Tight
  templates + reader unquote fix the three OBSERVED bugs at ~5% of the cost.
- **`/goal` + durable retry-loop:** cut from v3.8.0; revisit separately (validate `/clear`
  behaviour first; the verdict-probe `spec_ledger.py` subcommand doesn't exist yet).
- **session_epoch / expires_at-field / handoff_id:** speculative at this scale; no observed
  incident; `FRESH_SECS` + dead-pane check suffice.
- **git-commit signal / inotify / NATS / MCP / SQLite:** rejected (git-forbidden roles;
  daemon ceremony; nothing here needs them).

## Laptop note (one paragraph, not a tier system)
On a laptop with no cron/tmux, the relay simply isn't present; the operator hands off
manually. The template + reader fixes are inert-but-harmless there. No separate machinery.
